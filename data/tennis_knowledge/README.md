# 网球教练AI — 示例知识库

## 说明
这是一套模拟的网球教练知识库，用于在真实教练内容整理完成之前进行APP开发和测试。
所有内容均为示例数据，后续应替换为真实教练的教学经验。

## 目录结构

```
tennis_knowledge_base/
├── README.md                          ← 本文件
├── 技术教学/
│   ├── 01_正手击球.md                 ← 正手技术（初级→高级 + 问题诊断）
│   ├── 02_发球.md                     ← 发球技术（初级→中级 + 问题诊断）
│   ├── 03_反手击球.md                 ← 反手技术（双手/单手 + 问题诊断）
│   └── 04_步法与移动.md               ← 步法训练（基础→中级 + 训练方法）
├── 战术策略/
│   └── 01_单打战术.md                 ← 单打战术（初级→高级）
├── 训练计划/
│   └── 01_分级训练计划.md             ← 训练计划 + 体能训练建议
├── 常见问题/
│   ├── 01_常见问答.md                 ← 装备/学习/比赛/伤病FAQ
│   └── 02_示例问答对.md               ← 6组完整的用户-AI教练对话示例
└── 学员管理/
    （预留目录，用于后续存放学员进度追踪相关知识）
```

## 知识条目统计
- 技术教学条目：约40条（覆盖正手、反手、发球、步法）
- 问题诊断条目：约15条（含原因分析和纠正方法）
- 战术策略条目：约12条（覆盖初级到高级）
- 训练计划：3套（初级12周 + 中级周计划 + 体能训练）
- FAQ：约12个常见问题
- 示例问答对：6组完整对话

## 开发用途

### 用于RAG系统开发
1. 将这些 .md 文件作为数据源进行文档切分和向量化
2. 测试检索的准确性（用示例问答中的"用户提问"去查询，看返回的知识片段是否相关）
3. 调试System Prompt的效果（AI回答的语气和深度是否合适）

### 用于前端开发
1. 使用"示例问答对"作为聊天界面的模拟数据
2. 水平分级（2.0-4.5）可用于用户注册时的水平选择UI
3. 知识库的目录结构可用于"知识浏览"功能的导航设计

### 用于Fine-tuning准备
1. "示例问答对"展示了训练数据的目标格式
2. 每组问答都包含了用户信息、多轮对话、追问引导等要素
3. 可以参考这个格式来整理真实教练的问答数据

## 替换为真实数据时的注意事项
1. 保持文件名和目录结构一致，避免修改代码中的路径引用
2. 保持每个知识条目的格式（主题、适用水平、症状、诊断、纠正）不变
3. 新增的知识类别可以在对应目录下添加新的 .md 文件
4. 建议每次替换后重新运行向量化脚本，更新向量数据库
